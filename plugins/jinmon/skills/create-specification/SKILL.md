# create-specification

## いつ使うか

- jinmonのステップ1として呼ばれるとき
- 「仕様を固めて」と指示されたとき

## 実行手順

### 1. 要件を対話で詰める

`/grill-me` の要領で、以下が明確になるまで人間に質問する。

- 概要: 機能が何で何を達成したいのかを書く
- 背景: なぜこの機能が必要なのか
- 受入要件 (Given-When-Then形式)
- スコープ外(やらないこと)

### 2. 完了条件を確認する

以下を全て満たしてから次へ進むこと

- [ ] 機能の概要が1〜2文で説明できる
- [ ] なぜ必要かの動機・背景が明確になっている
- [ ] 受け入れ要件が全てGiven-When-Then形式で書かれている
- [ ] スコープ外(やらないこと)が明示されている
- [ ] 未解決の疑問点がゼロになっている
- [ ] 受け入れ要件からテストリストを書き起こせる状態になっている

### 3. 仕様ファイルを保存する

- 保存先はプロジェクトの規約(`CLAUDE.md`)に従うこと
- 規約がなければ `docs/spec/<feature-name>.md` に保存する
- プロエジェクトに仕様書のテンプレートがあれば従うこと、なければ[@SPEC_TEMPLATE.md](./SPEC_TEMPLATE.md)を使うこと

### 4. 完了報告

- 保存したファイルのパスを伝えて終了すること
