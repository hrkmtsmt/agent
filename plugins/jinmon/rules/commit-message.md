## コミットメッセージに含めるべき内容

- コミットメッセージの一行目はprefixをつけて、何をどの用に変更したかを記載すること
- コミットメッセージの二行目移行は、なぜそのような変更を行ったかを箇条書きで記載すること
- なにか参考にしたWeb上のドキュメントがあれば、再修行にリンクを記載すること

### Prefixの種類

- `break:`: 後方互換性を保てない破壊的変更が行われた場合に利用できる
- `feat:`: 新規機能を追加した場合に利用できる
- `fix:`: バグ修正や既存機能を変更した時やパフォーマンスの改善が行われた場合に利用できる
- `security:`: セキュリティに関する変更が加わった場合に利用できる
- `chore:`: ドキュメントの更新など実機能以外の変更が行われた場合に利用できる
- `refactor:`: 既存機能の振る舞いに変更がなく構造的な変更が行われた場合にのみ利用できる

### コミットメッセージのフォーマット

```md
feat: 何をどのように変更しました

- 理由や背景情報を箇条書きで簡潔に書く

ref: https://example.com
```
