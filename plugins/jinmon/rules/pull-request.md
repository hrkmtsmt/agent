# Pull Request

## PRの説明文

PRを作成するとき、説明文には以下の内容を必ず含めること

- **どんな変更か**: 変更の概要
- **なぜその変更が必要か**: 変更の動機・背景
- **受け入れ要件**: 仕様・動作確認の基準

## 仕様との整合性チェック(コードレビュー時)

- PRの説明文に受け入れ要件が記載されていない場合は、その旨をコメントすること
- PRの説明文に記載された受け入れ要件と、コードの変更内容が矛盾している場合は、その箇所を具体的に指摘すること

## PRフォーマット

基本的にはプロジェクトのPRフォーマットに従うこと
プロジェクトのPRフォーマットはリポジトリの`<ROOT>/.github/`ディレクトリにある
プロジェクトのPRフォーマットがない場合は、[PULL_REQUEST_TEMPLATE.md](../skills/create-pull-request/PULL_REQUEST_TEMPLATE.md) を使うこと
